# 3. Huvudverktyg

### &#x20;Enkla förklaringar

#### C# (C-Sharp)

**Vad det är:** Ett programmeringsspråk skapat av Microsoft, liknande Java men med moderna funktioner.

**Varför vi använder det:**

* Används av både frontend och backend = samma språk överallt
* Stark typning = färre buggar
* Stort community = lätt att hitta hjälp
* Bra för affärssystem

**Vardagsliknelse:** Som att prata samma språk hemma och på jobbet - enklare att kommunicera!

#### .NET 8

**Vad det är:** En plattform (som ett operativsystem för applikationer) som låter C#-kod köras på Windows, Mac och Linux.

**Varför vi använder det:**

* Senaste versionen = bästa prestanda
* Cross-platform = fungerar överallt
* Long-term support = stöd i flera år
* Gratis och open source

**Vardagsliknelse:** Som en motor som kan köra många olika bilar (applikationer), oavsett väg (operativsystem).

#### [ASP.NET](http://asp.net) Core Web API

**Vad det är:** Ett ramverk för att bygga "backend" - den del av applikationen som hanterar data, logik och säkerhet på servern.

**Varför vi använder det:**

* Skapar API:er (ett sätt för olika system att prata med varandra)
* Hanterar databaskommunikation
* Säkerhet (autentisering, kryptering)
* Prestanda och skalbarhet

**Vardagsliknelse:** Som en restaurangkök - gäster (frontend) beställer mat (data), köket (backend) lagar maten enligt regler (logik) och skickar ut den. "Tillagar man maten fel kan människor bli sjuka, som kan liknas med **dålig säkerhet i backend - användarnas data kan läcka ut eller stulits.**"

**Konkret i BetteRFlow:**

* Tar emot registreringar från webbsidan
* Validerar att email är korrekt format
* Hashar lösenord säkert
* Sparar data i databasen

#### Blazor Web App

**Vad det är:** Ett ramverk för att bygga "frontend" - det användaren ser och interagerar med i webbläsaren, men med C# istället för JavaScript.

**Varför vi använder det:**

* Samma språk som backend (C#)
* Komponentbaserad = återanvändbar kod
* Interaktiva gränssnitt utan JavaScript
* Server-side rendering = snabbare första laddning

**Vardagsliknelse:** Som ett fönster och kassadisk på restaurangen - det användaren ser och interagerar med, men det är byggt med samma “material” som köket, som du inte ser.

**Konkret i BetteRFlow:**

* Registreringsformulär som användare fyller i
* Inloggningssida
* Formulär för fastighetsdata
* Responsiv design (fungerar på mobil och dator)

#### MudBlazor

**Vad det är:** Ett bibliotek med färdiga designkomponenter (knappar, formulär, tabeller, dialoger) som följer Material Design.

**Varför vi använder det:**

* Professionellt utseende utan att designa från scratch
* Färdiga komponenter (textfält, knappar, tabeller)
* Responsiv design automatiskt
* Konsekvent användarupplevelse

**Vardagsliknelse:** Som IKEA-möbler för webbsidor - färdiga, snygga komponenter som bara behöver monteras ihop. Dock kan jag känna att IKEA just nu känns enklare. Det tog sin lilla tid att förstå sig på Blazor, när jag inte använt det tidigare.

**Konkret i BetteRFlow:**

Istället för att designa en knapp från noll:

* Använder MudBlazor-knapp som redan ser professionell ut
* Sparar tid för designarbete



#### Entity Framework Core

**Vad det är:** Ett ORM (Object-Relational Mapping) - översätter mellan C#-kod och databas utan att behöva skriva SQL-kod manuellt.

**Varför vi använder det:**

* Automatisk databaskod = snabbare utveckling
* Säkrare (skydd mot SQL-injection)
* Migrations = versionskontroll av databasstruktur
* Lätt att byta databas

**Vardagsliknelse:** Som en tolk mellan två personer som pratar olika språk - översätter C#-kod till databasspråk automatiskt, exempelvis svenska och teckenspråk.

**Konkret i BetteRFlow:**

**Traditionellt sätt - manuell SQL:**

```sql
"SELECT * FROM Users WHERE Email = 'test@test.com'"
```

**Problem med SQL-injection:**

Om användaren skriver `test@test.com' OR '1'='1` blir det:

```sql
SELECT * FROM Users WHERE Email = 'test@test.com' OR '1'='1'
```

**Vardagsliknelse:**

_"Ge mig kaffe om jag heter Maria ELLER om 1 = 1"_

Eftersom 1 alltid är lika med 1 → får personen kaffe oavsett namn! På samma sätt returneras **alla användare** från databasen, oavsett email.

#### **Med Entity Framework:**

```csharp
var user = context.Users.FirstOrDefault(u => u.Email == userInput);
```

**Vardagsliknelse:**

_"Ge mig kaffe **endast** om jag heter \[denna exakta text]: 'Maria ELLER om 1 = 1'"_

Ingen heter bokstavligen "Maria ELLER om 1 = 1" → inget kaffe! Attacken fungerar inte.

**Resultat:**

* Enklare kod
* Färre fel
* Automatiska säkerhetskontroller

#### SQLite & Azure SQL

**Vad det är:** Databashanteringssystem för att lagra och hämta data.

**SQLite** - Lokal utveckling:

* Fil-baserad databas (ingen server behövs)
* Lätt att versionshantera
* Perfekt för utveckling och tester

**Azure SQL** - Produktion:

* Molnbaserad databas
* Automatisk backup
* Skalbarhet och hög tillgänglighet

**Varför vi använder båda:**

* SQLite under utveckling = snabbt och enkelt
* Azure SQL i produktion = pålitligt och säkert

**Vardagsliknelse:** SQLite = Litet kassaskåp hemma för vardagsbruk Azure SQL = Bankvault för viktiga värdesaker

#### Git, GitBook & GitHub

**Vad det är:** Versionskontrollsystem för kod - sparar alla ändringar och låter flera utvecklare samarbeta.

**Varför vi använder det:**

* Historik över alla ändringar
* Kan ångra misstag
* Samarbeta med andra utvecklare
* Backup av kod
* Dokumentera exempelvis en rapport

**Vardagsliknelse:** Som "Spåra ändringar" i Word, fast mycket kraftfullare och för kod.

#### Swagger/OpenAPI

**Vad det är:** Automatisk dokumentation och testverktyg för API:er.

**Varför vi använder det:**

* Visar alla API-endpoints visuellt
* Testa API:et direkt i webbläsaren
* Automatiskt genererad från koden
* Ingen manuell dokumentation behövs

**Vardagsliknelse:** Som att åka till IKEA , där är det inte bara se, utan där får du även provsitta och känna vilken som fungerar bäst!

**Konkret i BetteRFlow:**

Gå till https://localhost:7007/swagger

* Se alla API-endpoints
* Testa registrering direkt
