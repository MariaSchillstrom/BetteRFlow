# 5. Scenario - Hur verktygen samarbetar

#### Situation:

En BRF-medlem ska registrera sig i systemet.

#### Traditionellt (utan moderna verktyg):

1. Skriva all HTML från scratch
2. Skriva all CSS för design
3. Skriva JavaScript för interaktivitet
4. Skriva SQL för databas manuellt
5. Hantera säkerhet själv
6. Testa allt manuellt

**1. MudBlazor (Frontend):**

* Färdigt formulär i 10 minuter
* Professionell design automatiskt
* Validering inkluderad

**2. Blazor (Frontend):**

* C#-kod istället för JavaScript
* Server-side rendering för snabbhet
* Automatisk säkerhet

**3. ASP.NET Core (Backend):**

* API-endpoint i 20 minuter
* Validering och säkerhet inkluderat
* Automatisk dokumentation via Swagger

**4. Entity Framework (Databas):**

* Databaskod automatisk
* Säker mot SQL-injection
* Migrations hanterar versioner

**5. Git (Versionskontroll):**

* Alla ändringar sparade
* Kan alltid gå tillbaka
* Backup automatisk

**Total tid: Inte lika lång tid som vid det traditionella! Återigen, allt är relativt!**

**Resultat:**

* Snabbare utveckling
* Högre kvalitet (färdiga, testade verktyg)
* Lägre kostnad
* Bättre säkerhet
